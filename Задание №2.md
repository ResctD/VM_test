## Часть 1. Структурный анализ
______
1. Уровни стандарт придуманы для создания профилей строгости, влияющих на бизнес-функции, аппаратную нагрузку. 
- Level 1 - базовая безопасная конфигурация, чтобы снизить поверхность атаки, перекрыв большую часть уязвимых точек узла. 
- Level 2 - строгая безопасная конфигурация, для настройки критических узлов организации. Перед эксплуатацией таких параметров рекомендуется производить предварительное функциональное тестирование и оценку производительности.

2. Поля проверок
- **№ проверки** (ID): Номер проверки
- **Наименование** (Title): Имя проверки и контекст
- **Профиль применимости** (Profile Applicability): для указания к какому уровню безопасной конфигурации относится текущая проверка
- **Описание** (Description): Объяснение что проверяется и какое значение рекомендуется как безопасное;
- **Обоснование** (Rationale): Объяснение почему этот параметр должен быть использован
- **Значение по умолчанию** (Default Value): значение, которое установлено по умолчанию
- **Влияние** (Impact): Информация по  возможным последствиям применения настройки. К каким утилитам/функциям пользователь получит ограничения, если параметр будет включен
- **Аудит** (Audit): Команды или скрипты или инструкции для проверки текущего состояния
- (**Remediation**): Инструкция по настройке параметра
- **Ссылки** (References): Адреса на документацию Microsoft или внешние источники
- **CIS Controls**: Ссылка на соответствующий контроль в CIS Controls и группу реализации.
- **Комментарий** (Note): Содержит дополнительную информацию в виде предупреждений, исключений.

Пример:
- **ID**: 1.1.4
- **Title**: Ensure 'Minimum password length' is set to '14 or more character(s)' (Automated
- **Profile**: Level 1
- **Description**:  This policy setting determines the least number of characters that make up a password for a user account. The recommended state for this setting is: 14 or more character(s).
- **Note**: In Windows Server 2016 and older versions of Windows Server, the GUI of the Local Security Policy (LSP), Local Group Policy Editor (LGPE) and Group Policy Management Editor (GPME) would not let you set this value higher than 14 characters. However, starting with Windows Server 2019, Microsoft changed the GUI to allow up to a 20 character minimum password length.
- **Note #2:** Password Policy settings (section 1.1) and Account Lockout Policy settings (section 1.2) must be applied via the Default Domain Policy GPO in order to be globally in effect on domain user accounts as their default behavior. If these settings are configured in another GPO, they will only affect local user accounts on the computers that receive the GPO. However, custom exceptions to the default password policy and account lockout policy rules for specific domain users and/or groups can be defined using Password Settings Objects (PSOs), which are completely separate from Group Policy and most easily configured using Active Directory Administrative Center.
- **Rationale**: Types of password attacks include dictionary attacks (which attempt to use common words and phrases) and brute force attacks (which try every possible combination of characters). Also, threat actors sometimes try to obtain the account database so they can use tools to discover the accounts and passwords.
- **Impact**: Requirements for extremely long passwords can decrease the security of an organization, because users might leave the information in an insecure location or lose it. If very long passwords are required, mistyped passwords could cause account lockouts and increase the volume of help desk calls. If your organization has issues with forgotten passwords due to password length requirements, consider teaching your users about passphrases, which are often easier to remember and, due to the larger number of character combinations, much harder to discover.
- **Audit**: Navigate to the UI Path articulated in the Remediation section and confirm it is set as prescribed.
- **Remediation**: To establish the recommended configuration via GP, set the following UI path to 14 or more character(s): 
	`Computer Configuration\Policies\Windows Settings\Security Settings\Account Policies\Password Policy\Enforce password history`
- **Default** Value: 7 characters on domain members. 0 characters on stand-alone servers..
- **References**: 
	- https://www.cisecurity.org/white-papers/cis-password-policy-guide/
	- https://learn.microsoft.com/en-us/windows/security/threat-protection/securitypolicy-settings/password-policy
	- GRID: MS-00000004
- **CIS Controls**:
	- v8, 5.2 Use Unique Passwords, IG 1, IG 2, IG 3
	- v7, 4.4 Use Unique Passwords, IG 2, IG 3
	- v7 16.2 Configure Centralized Point of Authentication





## Часть 2. Приоритизация и выбор 10 критичных проверок

### 18.6.7.6 Ensure 'Mandate the minimum version of SMB' is set to 'Enabled: 3.1.1' (Automated)
- **Снижение поверхности атаки**: Отключение устаревших и уязвимых версий SMB.
- **Защита от распространённых техник атак:** T1210
- **Почему в ТОП-10:** Одни из самых популярных атак EternalBlue, WannaCry эксплуатировали уязвимости SMBv1
- **Домен:** Administrative Templates (Computer)
- **Уровень** **CIS**: L1

### 2.3.7.6 Ensure 'Interactive logon: Number of previous logons to cache (in case domain controller is not available)' is set to '4 or fewer logon(s)' (Automated)
- **Снижение поверхности атаки:** Уменьшение количества сохранённых хешей паролей на узле.
- **Защита от распространённых техник атак: T1003:** Получение дампа учетных данных
- **Почему в ТОП-10:** Техника Т1003 является популярной по результатам различных исследований. Есть множество гайдов по использованию инструментов типа Mimikatz/NJRat и т.д., что уменьшает сложность и увеличивает популярность эксплуатации.
- **Домен:** Local Policies
- **Уровень CIS:** L2

### 1.2.2 Ensure 'Account lockout threshold' is set to '5 or fewer invalid logon attempt(s), but not 0' (Automated)
- **Снижение поверхности атаки:** Ограничения количества неудачных попыток входа на учётную запись с целью усложнить перебор паролей/учётных записей.
- **Защита от распространённых техник атак:** T1110: Метод перебора
- **Почему в ТОП-10:** Тактика атаки методами перебора все ещё одна из самых популярных. 
- **Домен:** Account Policies
- **Уровень CIS:** L1


### 2.3.11.6 Ensure 'Network security: LAN Manager authentication level' is set to 'Send NTLMv2 response only. Refuse LM & NTLM' (Automated)
- **Снижение поверхности атаки:** Отключение устаревших и не безопасных протоколов аутентификации.
- **Защита от распространённых техник атак:** T1550.002: Передача хеша (Pass the Hash)
- **Почему в ТОП-10:** Тактика очень популярная и одновременно не сложная для горизонтально перемещения за счёт гайдов по работе с готовыми инструментами Mimikatz, набора Impacket и т.д.
- **Домен:** Local Policies
- **Уровень CIS:** L1

### 2.3.1.3 Configure 'Accounts: Rename administrator account' (Automated)
- **Снижение поверхности атаки:** усложняет автоматизированные атаки, которые чаще всего написаны под стандартные учётные записи.
- **Защита от распространённых техник атак:** T1078: 
- **Почему в ТОП-10:** Увеличивает шансы на обнаружение подозрительных действий злоумышленника, так как в журналах событий будут видны стандартные учётные записи Administrators.
- **Домен**: Local Policies
- **Уровень CIS:** L1

### 2.3.17.1 Ensure 'User Account Control: Admin Approval Mode for the Built-in Administrator account' is set to 'Enabled' (Automated)
- **Снижение поверхности атаки**: предотвращение скрытого выполнения процессов с правами администратора.
- **Защита от распространённых техник атак**: T1548.002: Обход контроля учетных записей
- **Почему в ТОП-10:** Программы и скрипты будут запрашивать подтверждения повышения привилегий, что даёт возможность оставить действие вредоносного ПО и выполняет уведомление/предупреждение пользователя.
- **Домен:** Local Policies
- **Уровень CIS**: L1

### 18.10.42.10.3 Ensure 'Turn off real-time protection' is set to 'Disabled' (Automated)
- **Снижение поверхности атаки:** отключает возможность полностью деактивировать защиту от вредоносного ПО.
- **Защита от распространённых техник атак:** T1562.001 
- **Почему в ТОП-10:** Перед загрузкой любых инструментов на заражённый хост, крайне желательно отключить антивирусы на хосте.
- **Домен:** Administrative Templates (Computer)
- **Уровень CIS:** L2

### 18.10.42.6.1.2 Ensure 'Configure Attack Surface Reduction rules: Set the state for each ASR rule' is configured (Automated)
- **Снижение поверхности атаки:** Предотвращение атак через вредоносы, которые могли попасть через любые популярные рабочие инструменты, такие как почта, веб-сайты, флешки и т.д.
- **Защита от распространённых техник атак:** T1059 (Command and Scripting Interpreter), T1055 (Process Injection), **T1003 (Credential Dumping)**
- **Почему в ТОП-10:** Предоставляет защиту на уровне поведения, где уровень сигнатур может пропустить вредонос.
- **Домен:** Administrative Templates (Computer)
- **Уровень CIS:** L1

### 17.3.2 Ensure 'Audit Process Creation' is set to include 'Success' (Automated)
- **Снижение поверхности атаки:** Снижает в случае использования наложенный средств защиты информации, например SIEM, SOAR, EDR.
- **Защита от распространённых техник атак:** 
- **Почему в ТОП-10:** Журналирование на всех сетевый узлах необходимо для построения защищённой инфраструктуры.
- **Домен:** Advanced Audit Policy Configuration
- **Уровень CIS:** L1

### 18.10.94.2.1 Ensure 'Configure Automatic Updates' is set to 'Enabled' (Automated)
- **Снижение поверхности атаки:**  автоматическое устранение новых уязвимостей
- **Защита от распространённых техник атак:** **TA0001 (Initial Access)**, TA0008 (Lateral Movement)
- **Почему в ТОП-10:** Обновления безопасности затригивает все механизмы защиты хоста, поэтому они эффективны только при актуальности их версий и баз сигнатур
- *Домен:** Administrative Templates (Computer)
- **Уровень CIS:** L1


## Часть 3

### 18.6.7.6  Ensure 'Mandate the minimum version of SMB' is set to 'Enabled: 3.1.1' (Automated)
Эта настройка требует использования SMB 3.1.1. Она важна, потому что старые версии SMB 1.0 и 2.0 имеют известные уязвимости, которые активно эксплуатируются для распространения вредоносного ПО например, WannaCry и атак типа MITM. SMB 3.1.1 включает встроенное шифрование трафика, что защищает данные от перехвата, и обеспечивает аутентификацию с использованием современных криптографических алгоритмов.
Условие «прошел»: политика Mandate the minimum version of SMB установлена в Enabled, минимальная версия задана как 3.1.1.

### 2.3.7.6 Ensure 'Interactive logon: Number of previous logons to cache' is set to '4 or fewer logon(s)' (Automated)
Эта настройка ограничивает количество кешированных учётных данных, которые система сохраняет для входа в случае недоступности контроллера домена. Она важна для безопасности, потому что кешированные хеши паролей могут быть извлечены злоумышленником с помощью инструментов по типу mimikatz и использованы для горизонтального перемещения. Ограничение количества кешированных входов до 4 или менее снижает объем потенциально доступных злоумышленнику данных.
Условие «прошел»: параметр Number of previous logons to cache установлен в значение 4 или менее.

### 1.2.2 Ensure 'Account lockout threshold' is set to '5 or fewer invalid logon attempt(s), but not 0' (Automated)
Эта настройка определяет количество неудачных попыток входа, после которого учётная запись будет заблокирована. Нужна для защиты от брутфорс атак.
Условие «прошел»: параметр установлен в значение 5 или менее неудачных попыток, но не равен 0 (так как 0 отключает блокировку).

### 2.3.11.6 Ensure 'Network security: LAN Manager authentication level' is set to 'Send NTLMv2 response only. Refuse LM & NTLM' (Automated)
Эта настройка определяет, какие протоколы аутентификации могут использоваться в сети. Запрет устаревших протоколов LM и NTLM которые, позволяют злоумышленнику перехватывать или подделывать учётные данные. 
Условие «прошел»: параметр установлен в значение «Send NTLMv2 response only. Refuse LM & NTLM».

### 2.3.1.3 Configure 'Accounts: Rename administrator account' (Automated)
Эта настройка требует переименования встроенной учётной записи администратора, чтобы усложнить этап разведки.
Условие «прошел»: учётная запись администратора не Administrator.

### 2.3.17.1 Ensure 'User Account Control: Admin Approval Mode for the Built-in Administrator account' is set to 'Enabled' (Automated)
Эта настройка включает режим одобрения администратором для встроенной учётной записи Administrator, требуя подтверждения для всех действий, требующих повышенные привилегий, предотвращая выполнение вредоносного кода с правами администратора.
Условие «прошел»: политика "Admin Approval Mode for the Built-in Administrator account" установлена в "Enabled".

### 18.10.42.10.3 Ensure 'Turn off real-time protection' is set to 'Disabled' (Automated)
Эта настройка обеспечивает, то что защиту в реальном времени Microsoft Defender нельзя отключить. 
Условие «прошел»: политика "Turn off real-time protection" установлена в "Disabled"

### 18.10.42.6.1.2 Ensure 'Configure Attack Surface Reduction rules: Set the state for each ASR rule' is configured (Automated)
Эта настройка включает правила Attack Surface Reduction (ASR), которые блокируют запуск скриптов из Office, выполнение кода из email, кражу учётных данных из LSASS. 
Условие «прошел»: для каждого ASR-правила задано состояние — "1" (Включено) или "2" (Аудит).

### 17.3.2 Ensure 'Audit Process Creation' is set to include 'Success' (Automated)
Эта настройка включает аудит создания процессов, фиксируя каждый запуск программы в журнале безопасности. Она полезна для обнаружения вредоносного ПО, скриптов.
Условие «прошел»: аудит настроен на запись Success (успешные создания процессов).

### 18.10.94.2.1 Ensure 'Configure Automatic Updates' is set to 'Enabled' (Automated)
Эта настройка включает автоматическое получение обновлений безопасности, для своевременного закрытия новых уязвимостей. 
Условие «прошел»: политика Configure Automatic Updates установлена в Enabled с опцией 4 — Автоматическая загрузка и установка по расписанию.
